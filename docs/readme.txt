Root directory of the documentation of the project.

data_model - all artifacts related to the data model.
domain_analysis - all artifacts related to the domain model.
modelling_design - all artifacts related to OO modelling and design of the overall project, including architecture.
user_stories - all artifacts related to the modelling of each user story. There should be one folder per user story.
requirements_gathering - all artifacts related to requirements identified of the SW product to be developed.

In each folder there should be an "old" folder for deprecated artifacts.

Whenever possible, PlantUML should be used to create the artifacts.
